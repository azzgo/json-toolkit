## ADDED Requirements

### Requirement: 多语言类型生成
系统应当(SHALL)提供从 JSON 输入生成多种编程语言代码的功能，除了现有的 TypeScript 支持外，还应支持 Go、Java、Python (Pydantic)、Rust、Swift 和 C#。

#### Scenario: 从 JSON 生成 Go 结构体
- **当** 用户输入有效的 JSON 数据并选择 Go 作为目标语言
- **那么** 系统生成格式正确的 Go 结构体，包含正确的字段类型和 JSON 标签

#### Scenario: 从 JSON 生成 Python Pydantic 模型
- **当** 用户输入包含嵌套对象的 JSON 并选择 Python (Pydantic) 作为目标语言
- **那么** 系统生成带有适当类型提示和字段验证的 Pydantic BaseModel 类

#### Scenario: 处理复杂的 JSON 结构
- **当** 用户输入包含数组、嵌套对象、可选字段和混合类型的 JSON
- **那么** 系统为每种目标语言生成适当的代码构造 (数组作为切片/列表，可选字段作为指针/可选类型等)

### Requirement: 语言选择界面
系统应当(SHALL)提供直观的界面来选择目标编程语言并查看生成的代码。

#### Scenario: 带有实时预览的语言选择器
- **当** 用户更改选定的目标语言
- **那么** 系统立即重新生成新语言的代码并更新输出显示

#### Scenario: 多语言标签页
- **当** 用户想要比较不同语言的生成代码
- **那么** 系统允许通过标签页或分屏视图同时查看多个语言的输出

### Requirement: 代码输出管理
系统应当(SHALL)提供复制和下载各种格式生成代码的实用工具。

#### Scenario: 复制生成的代码
- **当** 用户点击生成代码的复制按钮
- **那么** 系统将代码复制到剪贴板并显示确认信息

#### Scenario: 下载为文件
- **当** 用户选择下载生成的代码
- **那么** 系统提供具有适当文件扩展名的下载 (.go, .java, .py, .rs, .swift, .cs)

### Requirement: 错误处理和验证
系统应当(SHALL)优雅地处理无效输入并提供有用的错误消息。

#### Scenario: 无效的 JSON 输入
- **当** 用户输入格式错误的 JSON
- **那么** 系统显示具体的错误消息，指出 JSON 解析问题以及行/列位置(如果可用)

#### Scenario: 不支持的 JSON 结构
- **当** JSON 包含在目标语言中无法表示的结构
- **那么** 系统提供解释限制的警告消息并提供替代方法

### Requirement: 性能和响应性
系统应当(SHALL)处理大型 JSON 输入并在代码生成期间提供响应反馈。

#### Scenario: 大型 JSON 处理
- **当** 用户输入大于 1MB 的 JSON
- **那么** 系统显示加载指示器并在 5 秒内生成代码或提供超时消息

#### Scenario: 实时验证
- **当** 用户在 JSON 输入区域中键入
- **那么** 系统提供即时验证反馈而不阻塞用户界面
